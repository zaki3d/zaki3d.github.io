Bitbucket Pipelines

Research, UI, UX • 2015-17

Pipelines is a Continuous Delivery tool that uses code as configuration and integrates directly within Bitbucket. It helps developers build, test, and deploy their code.

Bitbucket Pipelines was born to help small to medium-sized development teams quickly set up a Continuous Delivery pipeline for their projects. In this project, I worked with the PM, engineers, and my fellow Bitbucket designers in SF to envision, deliver and iteratively improve the product.

The birth of Pipelines

In late 2015, the Bamboo Cloud team pivoted from building what was essentially a hosted version of Bamboo Server into what they believed was a better way forward — Bitbucket Pipelines. If you're familiar with Bamboo, you'd know how powerful and feature-packed it was, but this came with a baggage. Setting up a pipeline in Bamboo was complicated and hard to maintain. Teams were required to sign up for a different service to run their build, and this added unnecessary overhead for most cloud teams.

Pipelines simplified that by using services like Docker as their build infrastructure, and using code as configuration to simplify pipeline maintenance and let the user focus on their code instead.
